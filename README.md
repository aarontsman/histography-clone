# Histography Clone

A local clone of [histography.io](https://histography.io) — an interactive canvas timeline of world history, built for self-hosting on a Raspberry Pi 5 (or any static web server).

## Features

- Interactive canvas timeline spanning the Big Bang to present day
- ~3,000 historical events fetched from Wikidata (wars, disasters, science, politics, film, music, art, sports)
- Wikipedia images and summaries loaded on demand
- Category filter chips with event counts
- Country filter with search-as-you-type dropdown
- Full-text search across event titles
- Hover tooltips and slide-in detail panel
- "Feeling Lucky" — jumps to a random event
- Material Design 3 dark theme (Roboto font, MD3 elevation surfaces, #D0BCFF primary)
- Log-scale timeline bar from Big Bang to present; linear zoom for the main view
- Momentum scrolling with pinch-to-zoom support

## Project Structure

```
histography-clone/

├── style.css           # Material Design 3 dark theme
├── app.js              # Canvas rendering, interaction, Wikipedia API
├── fetch_data.py       # Wikidata SPARQL fetcher (run once to populate data/)
├── deploy.sh           # One-command deploy to Raspberry Pi via rsync + nginx
└── data/
    └── events.json     # ~3,000 events (generated by fetch_data.py, not in repo)
```

## Quick Start (local preview)

Serve the directory with any static server:

```bash
npx serve .
# or
python3 -m http.server 8080
```

Open `http://localhost:8080` (or whichever port).

> `data/events.json` must exist. Either run `fetch_data.py` (see below) or copy the file from the RPi.

## Fetching Event Data

Requires Python 3 and the `requests` library:

```bash
pip install requests
python3 fetch_data.py
```

This queries Wikidata SPARQL across 36 categories and writes `data/events.json`. It takes roughly 10–20 minutes. If interrupted, resume with:

```bash
python3 fetch_data.py --resume
```

The fetcher respects Wikidata's rate limits with polite delays and exponential back-off on 429/timeout errors.

## Deploy to Raspberry Pi 5

```bash
bash deploy.sh pi@192.168.0.237
```

The script:
1. Creates `/var/www/histography` on the Pi
2. `rsync`s all web assets (excludes `data/`, `*.py`, `.git`)
3. Installs and configures nginx (no-cache headers for html/css/js; 10 min cache for json/images)
4. Reloads nginx

The `data/events.json` on the Pi is preserved across deploys. To regenerate it on the Pi:

```bash
ssh pi@192.168.0.237
sudo apt install python3-requests   # first time only
cd /var/www/histography
python3 fetch_data.py
```

## Tech Stack

- **Frontend**: Vanilla JS, Canvas 2D API — no frameworks, no build step
- **Data**: Wikidata SPARQL + Wikipedia REST API (`/api/rest_v1/page/summary`)
- **Rendering**: Pre-rendered sprite canvases per category; additive blending (`globalCompositeOperation: lighter`) for glow effect; twinkling via seeded sine waves at 20 fps
- **Timeline bar**: Log-scale mapping from −14 billion years to 2030; linear scale for the main viewport
- **Server**: nginx on Raspberry Pi OS (Debian bookworm)

## Data Format

Each event in `events.json`:

```json
{
  "id":        "Q12345",
  "year":      1969,
  "title":     "Moon landing",
  "desc":      "Short Wikidata description",
  "cat":       "science",
  "wiki":      "Apollo_11",
  "img":       "https://upload.wikimedia.org/…?width=400",
  "countries": ["United States"]
}
```

Categories: `war`, `disaster`, `science`, `politics`, `exploration`, `religion`, `film`, `music`, `art`, `sports`.
